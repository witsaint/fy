# 魔塔多模态模型配置指南

## 一、为什么使用多模态模型

### 传统方案 vs 多模态方案

**传统方案**：图片 → OCR文字识别 → 文本模型解析
- ❌ 两次API调用，速度慢
- ❌ OCR可能丢失表格结构信息
- ❌ 成本相对较高

**多模态方案**：图片 → 多模态模型直接识别
- ✅ 一次API调用，速度快
- ✅ 直接理解图片内容和表格结构
- ✅ 识别准确率更高
- ✅ 成本更低

## 五、参数调优

### Temperature（温度）

```bash
# 推荐值
OCR_TEMPERATURE=0.1      # 低温度，提高准确性
TEXT_TEMPERATURE=0.1     # 低温度，减少随机性

# 可选范围
0.0 - 0.2   # 结构化数据提取（推荐）
0.3 - 0.7   # 日常对话
0.8 - 1.5   # 创意写作
```

### Max Tokens（最大输出）

```bash
# 推荐值
OCR_MAX_TOKENS=2048      # OCR输出较长
TEXT_MAX_TOKENS=1024     # JSON输出较短

# 根据需求调整
512    # 简短输出
1024   # 标准输出（推荐）
2048   # 长文本输出
4096   # 超长输出
```

## 六、识别模式对比

### 模式1：直接识别（推荐）

```typescript
// 一步到位：图片 → JSON
const result = await recognizeImageWithStructure(image);

// 优点：
// ✅ 速度快（一次API调用）
// ✅ 准确率高（直接理解图片）
// ✅ 成本低（节省文本模型调用）

// 缺点：
// ⚠️ 依赖多模态模型性能
```

### 模式2：两步识别

```typescript
// 第一步：图片 → 文字
const text = await recognizeImageWithVisionModel(image);

// 第二步：文字 → JSON
const result = await parsePropertyFromText(text);

// 优点：
// ✅ 更灵活（可以查看中间文本）
// ✅ 便于调试（分步骤排查）

// 缺点：
// ⚠️ 速度慢（两次API调用）
// ⚠️ 成本高（两个模型）
```

## 十、常见问题

### Q1：如何切换模型？
**A**：修改`.env.local`中的`OCR_MODEL_ID`和`TEXT_MODEL_ID`即可，无需改代码。

### Q2：多模态模型支持哪些图片格式？
**A**：支持JPG、PNG、WEBP等常见格式，建议JPG格式，体积小速度快。

### Q3：如何提高识别准确率？
**A**：
1. 使用更高精度的模型（72B版本）
2. 降低temperature（如0.05）
3. 优化Prompt提示词
4. 确保图片清晰

### Q4：识别速度太慢怎么办？
**A**：
1. 切换到7B小模型
2. 减小max_tokens
3. 使用直接识别模式（跳过文本解析）

### Q5：可以本地部署模型吗？
**A**：可以，但需要：
- 高性能GPU（V100/A100）
- 下载模型文件（7B约14GB，72B约144GB）
- 搭建推理服务
- 建议使用魔塔云端API更方便

    return await recognizeImage(image);
  } catch (error) {
    retries++;
    if (retries === 3) throw error;
    await sleep(1000 * retries); // 指数退避
  }
}
