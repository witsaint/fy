# 数据字段规范文档

## 一、字段定义总览

本系统需要精准提取和管理的房源信息包含8个核心字段，每个字段都有严格的格式要求和验证规则。

| 序号 | 字段名称 | 字段代码 | 数据类型 | 是否必填 | 最大长度 |
|------|----------|----------|----------|----------|----------|
| 1 | 区域 | region | String | 是 | 50 |
| 2 | 楼盘 | building | String | 是 | 100 |
| 3 | 房号 | roomNumber | String | 是 | 50 |
| 4 | 面积 | area | String | 否 | 50 |
| 5 | 价格 | price | String | 否 | 100 |
| 6 | 物业费 | propertyFee | String | 否 | 100 |
| 7 | 佣金 | commission | String | 否 | 100 |
| 8 | 备注 | remarks | Text | 否 | 500 |

## 二、字段详细规范

### 字段1：区域（region）

#### 基本信息
- **字段含义**：房源所属的行政区域
- **数据类型**：String
- **是否必填**：是
- **最大长度**：50字符

#### 格式要求
1. **标准化名称**：必须使用行政区域全称，不能使用简称
   - ✅ 正确：瑶海区
   - ❌ 错误：瑶海

2. **允许值范围**（合肥市6个区）：
   - 瑶海区
   - 经开区
   - 蜀山区
   - 庐阳区
   - 包河区
   - 新站区

#### 识别规则
- AI识别时需匹配标准区域名称
- 识别到非标准名称时自动纠正：
  - "摇海区" → "瑶海区"
  - "经济开发区" → "经开区"
  - "蜀山" → "蜀山区"

#### 示例
```json
{
  "region": "瑶海区"  // 正确
}
```

### 字段2：楼盘（building）

#### 基本信息
- **字段含义**：房源所属的楼盘/建筑物名称
- **数据类型**：String
- **是否必填**：是
- **最大长度**：100字符

#### 格式要求
1. **保留完整名称**：与图片中显示的楼盘名称完全一致
2. **包含座栋信息**：如果图片中有座/栋/幢信息，需一并提取
3. **不做简化**：不省略"大厦"、"广场"等后缀

#### 识别规则
- 识别楼盘名称时保留原始格式
- 如果包含特殊符号（#、-、/），保持原样
- 座栋信息如果在楼盘名称中，则合并提取

#### 示例
```json
{
  "building": "万事通大厦"           // 完整名称
  "building": "尚泽大都会A座"        // 包含座信息
  "building": "中央广场写字楼"       // 保留后缀
  "building": "高科技园区1号楼"      // 保留编号
}
```

### 字段3：房号（roomNumber）

#### 基本信息
- **字段含义**：具体房间/单元的编号
- **数据类型**：String
- **是否必填**：是
- **最大长度**：50字符

#### 格式要求
1. **保留原始格式**：完全按照图片中的格式，不做任何修改
2. **包含特殊字符**：字母、数字、符号（#、-、/）全部保留
3. **大小写敏感**：字母大小写与原图保持一致

#### 识别规则
- 不对房号进行格式化
- 不自动补零（如"05"不要改成"5"）
- 不去除连接符

#### 示例
```json
{
  "roomNumber": "1005"              // 纯数字
  "roomNumber": "C#2517"            // 含字母和符号
  "roomNumber": "25栋2701-02"       // 含中文和连接符
  "roomNumber": "A座1203"           // 含座栋信息
  "roomNumber": "3-1-501"           // 多层级编号
}
```

### 字段4：面积（area）

#### 基本信息
- **字段含义**：房源的使用面积或工位数量
- **数据类型**：String
- **是否必填**：否
- **最大长度**：50字符

#### 格式要求
1. **数字+单位**：必须包含单位
2. **标准单位**：㎡（平方米）
3. **特殊情况**：
   - 如果原图单位是㎡，保持不变
   - 如果原图无单位，自动补充"㎡"
   - 如果是工位，保留原始描述（如"10个工位"）

#### 识别规则
- 提取数字和单位
- 无单位时默认补充"㎡"
- 非㎡单位（工位、间）保留原文

#### 示例
```json
{
  "area": "252㎡"                   // 标准格式
  "area": "10个工位"                // 非标准单位
  "area": "150平方米"               // 保留完整单位
  "area": "85㎡左右"                // 保留修饰词
  "area": "无"                      // 空值标准化
}
```

#### 数据规整规则
```javascript
// 示例：面积字段规整逻辑
function normalizeArea(rawArea) {
  if (!rawArea) return "无";
  
  // 如果已有单位，保持不变
  if (rawArea.includes('㎡') || rawArea.includes('工位') || rawArea.includes('平方米')) {
    return rawArea;
  }
  
  // 纯数字，自动补充㎡
  if (/^\d+$/.test(rawArea)) {
    return rawArea + '㎡';
  }
  
  return rawArea;
}
```

### 字段5：价格（price）

#### 基本信息
- **字段含义**：房源的租金价格
- **数据类型**：String
- **是否必填**：否
- **最大长度**：100字符

#### 格式要求
1. **数字+单位**：必须保留完整的价格和计价单位
2. **常见格式**：
   - 元/㎡/天
   - 元/㎡/月
   - 元/月
   - 元/年
3. **特殊情况**：
   - "面议"：保留原文
   - "价格面谈"：保留原文
   - 无价格信息：标注"无"

#### 识别规则
- 提取完整的价格表达式
- 保留所有单位信息
- 不进行单位换算

#### 示例
```json
{
  "price": "42元/㎡/天"             // 标准格式
  "price": "3300元/月"              // 月租
  "price": "面议"                   // 价格待定
  "price": "2.5元/㎡/天起"          // 含起始价
  "price": "50-60元/㎡/月"          // 价格区间
  "price": "无"                     // 空值
}
```

### 字段6：物业费（propertyFee）

#### 基本信息
- **字段含义**：物业管理费用
- **数据类型**：String
- **是否必填**：否
- **最大长度**：100字符

#### 格式要求
1. **标准格式**：数字+单位（元/㎡/月）
2. **特殊情况处理**：
   - 含在租金内：标注"含在租金内"
   - 免物业费：标注"免物业费"

#### 识别规则
- 识别关键词："物业费"、"管理费"
- 识别"含"、"包含"等关键词，标注为"含在租金内"
- 提取数字和单位

#### 示例
```json
{
  "propertyFee": "3.5元/㎡/月"       // 标准格式
  "propertyFee": "含在租金内"         // 已包含
  "propertyFee": "免物业费"           // 免费
  "propertyFee": "5元/平米/月"        // 其他单位表达
  "propertyFee": "无"                 // 无信息
}
```

### 字段8：备注（remarks）

#### 基本信息
- **字段含义**：其他补充信息
- **数据类型**：Text
- **是否必填**：否
- **最大长度**：500字符

## 三、空值处理规范

### 空值标准化规则
1. **必填字段**（区域、楼盘、房号）：
   - 不允许为空
   - 识别失败时标记为"识别失败"
   - 需要人工补充

2. **选填字段**（面积、价格、物业费、佣金、备注）：
   - 无值时统一标注"无"
   - 不使用null、undefined、空字符串
   - 前端展示"无"时使用浅灰色

### 空值识别场景
```json
{
  "region": "瑶海区",
  "building": "万事通大厦",
  "roomNumber": "1005",
  "area": "无",              // 图片中无面积信息
  "price": "面议",           // 价格待定
  "propertyFee": "含在租金内", // 特殊情况，非空值
  "commission": "无",         // 无佣金信息
  "remarks": "精装修"         // 有备注信息
}
```

## 四、数据验证规则

### 4.1 必填字段验证
```javascript
// 验证必填字段
function validateRequired(property) {
  const errors = [];
  
  if (!property.region || property.region === '识别失败') {
    errors.push('区域为必填项');
  }
  
  if (!property.building || property.building === '识别失败') {
    errors.push('楼盘为必填项');
  }
  
  if (!property.roomNumber || property.roomNumber === '识别失败') {
    errors.push('房号为必填项');
  }
  
  return errors;
}
```

### 4.2 区域标准化验证
```javascript
// 验证区域是否在允许列表中
const VALID_REGIONS = ['瑶海区', '经开区', '蜀山区', '庐阳区', '包河区', '新站区'];

function validateRegion(region) {
  if (!VALID_REGIONS.includes(region)) {
    return {
      valid: false,
      message: `区域"${region}"不在标准列表中，请选择：${VALID_REGIONS.join('、')}`
    };
  }
  return { valid: true };
}
```

### 4.3 长度验证
```javascript
// 验证字段长度
const MAX_LENGTHS = {
  region: 50,
  building: 100,
  roomNumber: 50,
  area: 50,
  price: 100,
  propertyFee: 100,
  commission: 100,

function validateLength(field, value) {
  const maxLength = MAX_LENGTHS[field];

    return {
      valid: false,
      message: `${field}长度不能超过${maxLength}字符`
    };
  }
  return { valid: true };
}
```

## 五、数据规整流程

### 5.1 识别后规整流程
```
OCR识别原始文本
       ↓
AI提取结构化字段
       ↓
字段值规整处理
       ↓
├─ 区域标准化（匹配6个标准区域）
├─ 面积单位补充（无单位时添加㎡）
├─ 空值统一标注（选填字段无值标注"无"）
└─ 去除多余空格和标点
       ↓
数据验证
       ↓
├─ 必填字段检查
├─ 区域有效性检查
└─ 长度限制检查
       ↓
存入数据库
```

### 5.2 规整函数示例
```javascript
// 完整的数据规整函数
function normalizeProperty(rawProperty) {
  return {
    region: normalizeRegion(rawProperty.region),
    building: rawProperty.building?.trim() || '识别失败',
    roomNumber: rawProperty.roomNumber?.trim() || '识别失败',
    area: normalizeArea(rawProperty.area),
    price: normalizePrice(rawProperty.price),
    propertyFee: normalizePropertyFee(rawProperty.propertyFee),
    commission: rawProperty.commission?.trim() || '无',
    remarks: rawProperty.remarks?.trim() || '无'
  };
}

// 区域标准化
function normalizeRegion(region) {
  if (!region) return '识别失败';
  
  const regionMap = {
    '摇海区': '瑶海区',
    '瑶海': '瑶海区',
    '经济开发区': '经开区',
    '经开': '经开区',
    '蜀山': '蜀山区',
    '庐阳': '庐阳区',
    '包河': '包河区',
    '新站': '新站区'
  };
  
  return regionMap[region] || region;
}

// 面积标准化
function normalizeArea(area) {
  if (!area) return '无';
  if (area.includes('㎡') || area.includes('工位')) return area;
  if (/^\d+$/.test(area)) return area + '㎡';
  return area;
}

// 价格标准化
function normalizePrice(price) {
  if (!price) return '无';
  if (price.includes('面议') || price.includes('面谈')) return '面议';
  return price;
}

// 物业费标准化
function normalizePropertyFee(fee) {
  if (!fee) return '无';
  if (fee.includes('含') || fee.includes('包含')) return '含在租金内';
  if (fee.includes('免')) return '免物业费';
  return fee;
}
```

## 七、数据质量控制

### 7.1 识别置信度评分
```javascript
// 计算识别置信度
function calculateConfidence(property) {
  let score = 0;
  
  // 核心字段（每个占0.15分，共0.6分）
  if (property.region && property.region !== '识别失败') score += 0.15;
  if (property.building && property.building !== '识别失败') score += 0.15;
  if (property.roomNumber && property.roomNumber !== '识别失败') score += 0.15;
  if (property.price && property.price !== '无') score += 0.15;
  
  // 其他字段（每个占0.1分，共0.4分）
  if (property.area && property.area !== '无') score += 0.1;
  if (property.propertyFee && property.propertyFee !== '无') score += 0.1;
  if (property.commission && property.commission !== '无') score += 0.1;
  if (property.remarks && property.remarks !== '无') score += 0.1;
  
  return score;
}
```

### 7.2 质量等级划分
- **高质量**（置信度≥0.9）：所有核心字段+大部分选填字段完整
- **中质量**（0.7≤置信度<0.9）：核心字段完整，部分选填字段缺失
- **低质量**（0.5≤置信度<0.7）：核心字段部分缺失
- **需人工检查**（置信度<0.5）：核心字段大量缺失，需人工补充

### 7.3 前端质量标记
```javascript
// 根据置信度显示不同标记
function getQualityBadge(confidence) {
  if (confidence >= 0.9) {
    return { text: '高质量', color: 'green' };
  } else if (confidence >= 0.7) {
    return { text: '中等', color: 'blue' };
  } else if (confidence >= 0.5) {
    return { text: '待完善', color: 'orange' };
  } else {
    return { text: '需人工检查', color: 'red' };
  }
}
```

## 八、常见问题FAQ

### Q1：如何处理图片中没有某个字段的情况？
**A**：选填字段（面积、价格、物业费、佣金、备注）统一标注"无"；必填字段（区域、楼盘、房号）标注"识别失败"，需人工补充。

### Q2：区域名称识别错误怎么办？
**A**：系统会自动进行区域名称标准化，常见错误会自动纠正。如果纠正后仍不在6个标准区域中，则标注"识别失败"，需人工选择。

### Q3：价格单位不统一怎么处理？
**A**：完整保留原始价格表达式，不进行单位换算。不同单位格式（元/㎡/天、元/月）均保持原样。

### Q4：面积字段有时是工位数量，如何处理？
**A**：保留原始描述。如果是"10个工位"，不要转换成面积；如果是纯数字，自动补充"㎡"单位。

### Q5：备注字段包含多条信息，如何组织？
**A**：多条信息用逗号分隔，按重要性排序（装修情况>位置信息>其他）。

## 十、数据字典

### 数据库表结构（参考）
```sql
CREATE TABLE properties (
  -- 主键
  id VARCHAR(36) PRIMARY KEY COMMENT '房源ID',
  file_id VARCHAR(36) NOT NULL COMMENT '关联的图片文件ID',
  
  -- 核心字段（必填）
  region VARCHAR(50) NOT NULL COMMENT '区域',
  building VARCHAR(100) NOT NULL COMMENT '楼盘',
  room_number VARCHAR(50) NOT NULL COMMENT '房号',
  
  -- 选填字段
  area VARCHAR(50) DEFAULT '无' COMMENT '面积',
  price VARCHAR(100) DEFAULT '无' COMMENT '价格',
  property_fee VARCHAR(100) DEFAULT '无' COMMENT '物业费',
  commission VARCHAR(100) DEFAULT '无' COMMENT '佣金',
  remarks TEXT COMMENT '备注',
  
  -- 质量相关
  confidence DECIMAL(3,2) COMMENT '识别置信度',
  
  -- 时间戳
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  -- 索引
  INDEX idx_region (region),
  INDEX idx_building (building),
  INDEX idx_confidence (confidence)
) COMMENT '房源信息表';
```
