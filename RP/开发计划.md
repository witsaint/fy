# 房源图片信息汇总Web应用 - 开发计划

## 一、项目概述

**项目名称**：房源图片信息汇总工具  
**项目类型**：Web应用  
**开发模式**：全栈一体化（Next.js）  
**技术栈**：
- 框架：Next.js 14+ (App Router) + TypeScript
- UI组件库：Ant Design / shadcn/ui
- 数据存储：无数据库，内存临时存储（批次处理）
- OCR服务：第三方API（百度OCR/腾讯OCR）
- AI解析：OpenAI/Claude/通义千问 API
- Excel导出：xlsx 库（前端生成）

## 二、核心功能模块划分

### 模块1：图片上传模块
- **前端组件**：文件上传组件、拖拽上传、图片预览
- **API路由**：文件接收、格式验证、Base64转换

### 模块2：AI信息识别模块
- **前端组件**：识别进度展示、结果反馈
- **API路由**：OCR图像识别、AI结构化解析、数据规整
- **Server Actions**：批量识别任务处理

### 模块3：表格汇总模块
- **前端组件**：数据表格展示、筛选、编辑、删除
- **客户端状态**：React State管理（内存中），无需后端存储

### 模块4：数据导出模块
- **前端组件**：导出按钮、Excel生成
- **前端库**：使用xlsx库在浏览器端生成Excel

## 四、技术难点与解决方案

### 难点1：OCR识别准确率
- **解决方案**：选用成熟OCR服务（如百度OCR、腾讯OCR、阿里OCR）
- **备选方案**：支持手动编辑修正识别结果

### 难点2：表格信息结构化提取
- **解决方案**：使用AI大模型（GPT/Claude）进行二次解析
- **流程**：OCR识别文字 → AI提取结构化字段 → 格式规整

### 难点3：无数据库的数据管理
- **解决方案**：使用React State管理当前批次数据
- **特点**：数据仅在当前会话有效，刷新页面数据清空
- **优势**：无需数据库配置，部署简单

### 难点4：批量图片处理性能
- **解决方案**：
  - Next.js API路由处理单张图片识别
  - 前端使用Promise.all并发处理（限制并发数5-10）
  - 使用React State实时更新进度
- **优化**：限制并发数量，避免API限流

## 五、数据流程图

```
用户选择图片（浏览器端）
    ↓
前端验证（格式/大小/数量）
    ↓
转换为Base64并上传到API路由
    ↓
API路由调用OCR服务识别
    ↓
API路由调用AI解析结构化数据
    ↓
返回JSON数据到前端
    ↓
前端保存到React State（内存）
    ↓
实时展示在表格中
    ↓
用户编辑/筛选数据（客户端操作）
    ↓
点击导出：前端使用xlsx生成Excel文件
    ↓
浏览器自动下载Excel文件
```

## 八、风险预警

### 技术风险
1. **OCR服务稳定性**：建议准备多个OCR服务商备选
2. **识别准确率不达标**：需要AI二次解析和人工修正机制
3. **复杂表格格式**：需要支持多种表格布局的识别

### 成本风险
1. **OCR API调用费用**：评估使用量，选择性价比方案
2. **AI模型调用费用**：考虑本地模型或免费额度

### 时间风险
1. **OCR调试时间**：需要大量测试样本优化识别效果
2. **格式兼容性**：不同来源的表格图片差异大
