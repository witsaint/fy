# 图片文本翻译替换 - 需求文档（V1.0）

## 一、项目定位

**一句话描述：** 上传图片 → 服务端识别图片中的中文 → 翻译成目标语言 → 替换原图文字 → 返回给用户下载。

**架构：**
- 框架：NextJS（前后端一体，无独立后端服务）
- 无数据库、无文件存储、无用户系统
- 图片在服务端内存中处理完后直接返回，不落盘
- AI 能力：魔塔 `qwen-vl-plus`（识别）+ `qwen-turbo`（翻译）

```
用户上传图片
     ↓
POST /api/translate（图片 base64 + 目标语言）
     ↓
qwen-vl-plus 识别图片中的中文文本及位置
     ↓
qwen-turbo 翻译成目标语言
     ↓
Sharp + Canvas 在原图上替换文字
     ↓
返回处理后的图片 base64 给前端
     ↓
用户预览 / 下载
```

## 二、功能需求

### 2.1 上传

- 支持拖拽和点击上传
- 支持格式：JPG、PNG、WebP，单张≤10MB，最多20张
- 上传后显示缩略图列表

### 2.2 语言设置

- 选择目标语言（默认英语）
- 支持：英语、日语、韩语、西班牙语、法语、德语、俄语、泰语、越南语、印度尼西亚语

### 2.3 处理（支持批量）

- 点击「开始翻译」触发批量处理
- **接口设计：1个接口，每次处理1张图片**；前端负责批量调度，最多3张并发，其余排队
- 每张图片实时显示独立状态：待处理 / 处理中 / 已完成 / 失败
- 单张失败不影响其他图片继续处理，失败图片可单独重试

### 2.4 结果展示与下载

- 处理完成后，图片卡片显示翻译后的缩略图
- 支持单张下载
- 支持批量打包 ZIP 下载（前端完成，无需服务端）

## 三、技术约束

| 项目 | 说明 |
| --- | --- |
| 框架 | NextJS 14（App Router） |
| 包管理器 | pnpm |
| 数据库 | ❌ 无 |
| 文件存储 | ❌ 无（图片不落盘） |
| 用户系统 | ❌ 无 |
| 历史记录 | ❌ 无 |
| AI 服务 | 魔塔 ModelScope API |
| 图片处理 | Sharp + canvas（Node.js） |
| 批量方式 | 1个接口，前端逐张调用，最多3并发 |
