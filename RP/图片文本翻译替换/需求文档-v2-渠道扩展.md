# 需求文档 v2 — 多渠道图生图翻译

## 背景

现有功能：上传广告图片 → `qwen-image-edit` 识别中文并替换为目标语言。

新需求：引入**渠道**概念，当前先支持**火山引擎**。处理流程改为三步分离，使不同渠道可灵活插拔：

1. 识别图片中的非商品文字（字符识别，与渠道无关）
2. 将识别到的文字翻译为目标语言（与渠道无关）
3. 通过**所选渠道**的多模态大模型进行图生图，将翻译后的文字替换回原图

---

## 渠道定义

| 渠道 Key | 渠道名称 | 当前状态 |
|---|---|---|
| `volcengine` | 火山引擎 | ✅ 支持 |
| `qwen` | 通义千问 / DashScope | 🔜 预留 |

## 页面改动

### 新增「渠道选择」下拉

- 位置：在「目标语言」选择器旁边
- 选项：火山引擎（默认）
- 后续可追加其他渠道，无需改页面结构

### 调用时携带渠道参数

前端请求 `/api/translate` 时，Body 新增 `channel` 字段：

```json
{
  "id": "xxx",
  "base64": "...",
  "format": "jpeg",
  "targetLang": "en",
  "channel": "volcengine"
}
```

## 后端接口改动

### `POST /api/translate`

新增入参 `channel`（默认 `volcengine`）。

处理流程：

```
原图 base64
  │
  ▼
[Step 1] OCR 识别非商品文字
  使用 qwen-vl-plus 或魔塔多模态，提取广告文案文字（排除商品名、Logo、价格等）
  返回：识别到的文字列表
  │
  ▼
[Step 2] 批量翻译
  使用 qwen-turbo 或同类文本模型，将文字列表翻译为 targetLang
  返回：{ original, translated }[]
  │
  ▼
[Step 3] 渠道图生图
  根据 channel 分发到对应渠道处理器：
  - volcengine → 火山引擎多模态图生图 API
  构造 prompt：原图 + "将 [原文] 替换为 [译文]，其他内容不变"
  返回：处理后图片 base64
```

### 响应结构（不变）

```json
{
  "id": "xxx",
  "success": true,
  "translatedBase64": "...",
  "texts": [
    { "original": "限时特惠", "translated": "Limited Time Offer" }
  ]
}
```

## 火山引擎接入

### 使用模型

火山引擎视觉理解 + 图像生成能力，具体模型待确认（参考：`doubao-seedream` 系列或视觉编辑模型）。

### 鉴权

| 环境变量 | 说明 |
|---|---|
| `VOLCENGINE_ACCESS_KEY` | 火山引擎 Access Key |
| `VOLCENGINE_SECRET_KEY` | 火山引擎 Secret Key |
| `VOLCENGINE_ENDPOINT_ID` | 推理接入点 ID（模型部署后获取） |

### 调用方式

火山引擎图生图 API 为异步任务模式（类似魔塔）：
1. POST 提交任务
2. 轮询查询任务状态
3. 任务完成后下载图片转 base64 返回
