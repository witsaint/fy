# 前端需求文档（Next.js版本）

## 一、技术栈

### 核心技术
- **框架**：Next.js 14+ (App Router) + React 18
- **语言**：TypeScript
- **UI组件库**：Ant Design 5.x
- **样式方案**：Tailwind CSS + CSS Modules
- **状态管理**：React State (useState)
- **Excel导出**：xlsx（前端生成）
- **HTTP客户端**：fetch（原生）

### 特点
- **全栈一体**：前后端代码在同一项目中
- **无需数据库**：数据存储在React State中
- **批次处理**：每次上传识别为一个独立批次

## 三、核心功能模块

### 3.1 图片上传模块

#### 功能点清单
1. **上传按钮**
   - 大按钮样式，醒目易点
   - 文字："上传房源图片"
   - 图标：文件夹图标
   - 点击后弹出文件选择框

2. **拖拽上传**
   - 支持拖拽图片到上传区域
   - 拖拽时显示虚线边框和提示文字
   - 拖拽完成自动开始上传

3. **格式和大小验证**
   - 仅接受`.jpg`、`.jpeg`、`.png`格式
   - 单个文件≤10MB
   - 超过限制时弹出提示："仅支持JPG/PNG格式，单张≤10MB"

4. **批量上传**
   - 一次最多选择20张图片
   - 超过20张时提示："最多上传20张图片"

5. **上传进度**
   - 显示上传进度条
   - 文字提示："正在上传 X/Y 张图片"
   - 上传完成提示："XX张图片上传成功，开始识别"

6. **错误处理**
   - 网络错误：提示"上传失败，请检查网络连接"
   - 服务器错误：提示"服务器繁忙，请稍后重试"

#### 技术实现
```javascript
// 推荐库：react-dropzone / ant-design Upload组件
<Upload
  multiple
  accept=".jpg,.jpeg,.png"
  maxCount={20}
  beforeUpload={validateFile}
  onChange={handleUpload}
  showUploadList={false}
>
  <Button size="large" icon={<UploadOutlined />}>
    上传房源图片
  </Button>
</Upload>
```

### 3.3 识别进度模块

#### 功能点清单
1. **识别状态展示**
   - 识别中：显示进度条和提示"正在识别XX张图片，请稍等"
   - 识别完成：提示"识别完成，共提取XX条房源信息"
   - 识别失败：提示"图片模糊/无表格信息，无法识别"

2. **实时进度更新**
   - 通过WebSocket接收后端进度
   - 显示当前进度：3/10张已识别
   - 显示识别速度：预计剩余XX秒

3. **重新识别功能**
   - 按钮："重新识别"
   - 点击后清空当前结果，重新调用识别接口
   - 识别中禁用按钮，防止重复点击

#### 技术实现
```javascript
// WebSocket连接
const ws = new WebSocket('ws://localhost:3000/ws/progress');
ws.onmessage = (event) => {
  const { current, total } = JSON.parse(event.data);
  setProgress({ current, total });
};
```

### 3.6 数据导出模块

#### 功能点清单
1. **导出按钮**
   - 位置：表格下方居中
   - 样式：大按钮，蓝色背景
   - 文字："💾 导出Excel"
   - 点击触发导出

2. **导出逻辑**
   - 前端调用后端API `/api/export/excel`
   - 后端返回文件下载URL
   - 前端触发浏览器下载

3. **导出范围**
   - 如果当前有筛选条件，导出筛选后的数据
   - 如果没有筛选，导出全部数据

4. **导出反馈**
   - 点击后显示"正在导出，请稍等"
   - 导出成功提示"Excel已保存至：桌面"（Electron环境）
   - 导出失败提示错误信息

5. **文件命名**
   - 格式：`房源汇总表_YYYYMMDD.xlsx`
   - 示例：`房源汇总表_20260215.xlsx`

#### 技术实现
```javascript
const handleExport = async () => {
  try {
    setExporting(true);
    const response = await axios.post('/api/export/excel', {
      region: currentRegion // 当前筛选条件
    });
    
    // 触发下载
    const link = document.createElement('a');
    link.href = response.data.downloadUrl;
    link.download = response.data.fileName;
    link.click();
    
    message.success('Excel已保存至桌面');
  } catch (error) {
    message.error('导出失败，请重试');
  } finally {
    setExporting(false);
  }
};
```

## 四、状态管理

### 4.1 全局状态（推荐使用Zustand）

```javascript
// store.js
import create from 'zustand';

const useStore = create((set) => ({
  // 上传的图片列表
  images: [],
  setImages: (images) => set({ images }),
  
  // 房源数据列表
  properties: [],
  setProperties: (properties) => set({ properties }),
  
  // 当前筛选的区域
  currentRegion: '',
  setCurrentRegion: (region) => set({ currentRegion: region }),
  
  // 识别进度
  recognizeProgress: { current: 0, total: 0 },
  setRecognizeProgress: (progress) => set({ recognizeProgress: progress }),
  
  // 加载状态
  loading: {
    uploading: false,
    recognizing: false,
    exporting: false
  },
  setLoading: (key, value) => set((state) => ({
    loading: { ...state.loading, [key]: value }
  }))
}));
```

## 五、用户体验优化

### 5.1 加载状态
- 所有异步操作（上传、识别、导出）显示加载动画
- 使用Spin组件包裹内容区域
- 按钮添加loading状态，防止重复点击

### 5.2 提示反馈
- 成功操作：绿色提示框，3秒自动消失
- 错误操作：红色提示框，5秒自动消失
- 警告信息：橙色提示框，4秒自动消失
- 使用Ant Design的message/notification组件

### 5.3 空状态处理
- 未上传图片时：显示"请上传房源图片开始识别"
- 识别后无数据：显示"未识别到有效房源信息"
- 筛选后无数据：显示"该区域暂无房源信息"

### 5.4 响应式设计
- 适配不同屏幕尺寸（最小支持1280x720）
- 表格横向滚动（内容超出时）
- 按钮在小屏幕上堆叠排列

### 5.5 快捷键支持
- `Ctrl/Cmd + U`：打开上传文件
- `Ctrl/Cmd + E`：导出Excel
- `Ctrl/Cmd + R`：重新识别
- `ESC`：关闭弹窗

## 六、性能优化

### 6.1 图片优化
- 缩略图压缩：上传后生成120x120缩略图
- 懒加载：图片列表使用虚拟滚动
- 预览图按需加载：点击时才加载原图

### 6.2 表格优化
- 虚拟滚动：数据量大时使用虚拟列表（react-window）
- 分页加载：每次只渲染20条数据
- 防抖搜索：搜索框输入防抖500ms

### 6.3 代码优化
- 组件懒加载：使用React.lazy分割代码
- 状态更新优化：使用useMemo/useCallback避免不必要渲染
- 防抖节流：用户操作频繁的地方添加防抖/节流

## 七、错误处理

### 7.1 网络错误
- 接口调用失败：显示"网络连接失败，请重试"
- 超时处理：30秒超时，提示用户
- 自动重试：关键接口失败自动重试3次

### 7.2 数据错误
- 后端返回错误：显示具体错误信息
- 数据格式错误：前端校验，提示用户修正
- 空数据处理：显示友好的空状态页面

### 7.3 用户操作错误
- 文件格式错误：上传前校验，拒绝上传
- 表单验证错误：高亮错误字段，显示错误提示
- 重复提交：按钮loading状态防止重复点击

## 十一、关键代码示例

### API服务封装
```javascript
// services/api.js
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:3000/api',
  timeout: 30000
});

// 上传图片
export const uploadImages = (files) => {
  const formData = new FormData();
  files.forEach(file => formData.append('files', file));
  return api.post('/upload', formData);
};

// 触发识别
export const recognizeImages = (fileIds) => {
  return api.post('/recognize', { fileIds });
};

// 获取房源列表
export const getProperties = (params) => {
  return api.get('/properties', { params });
};

// 更新房源
export const updateProperty = (id, data) => {
  return api.put(`/properties/${id}`, data);
};

// 删除房源
export const deleteProperty = (id) => {
  return api.delete(`/properties/${id}`);
};

// 导出Excel
export const exportExcel = (params) => {
  return api.post('/export/excel', params);
};

// 清空数据
export const clearData = () => {
  return api.post('/data/clear');
};
```

### Electron主进程
```javascript
// electron/main.js
const { app, BrowserWindow } = require('electron');
const path = require('path');

function createWindow() {
  const win = new BrowserWindow({
    width: 1280,
    height: 800,
    title: '房源信息汇总工具',
    webPreferences: {
      nodeIntegration: true,
      contextIsolation: false
    }
  });

  // 开发环境
  if (process.env.NODE_ENV === 'development') {
    win.loadURL('http://localhost:5173');
  } else {
    // 生产环境
    win.loadFile(path.join(__dirname, '../dist/index.html'));
  }
}

app.whenReady().then(createWindow);
```
